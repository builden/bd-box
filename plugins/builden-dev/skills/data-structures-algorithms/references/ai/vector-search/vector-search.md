# 向量搜索

## 一句话定义

在高维空间中基于向量相似度进行检索的技术，是 AI 应用的核心基础设施。

## 为什么好

**优点：**

- 支持语义搜索
- 处理非结构化数据
- 相似度检索高效

**缺点：**

- 维度灾难
- 精确搜索困难
- 内存占用大

## 适用场景

- **语义搜索**：自然语言查询
- **推荐系统**：相似物品
- **图像检索**：以图搜图
- **RAG**：知识库检索

## 核心概念

### 1. 向量表示

将文本、图像等转换为向量：

- **词向量**：Word2Vec、BERT
- **图像向量**：ResNet、VGG
- **句子向量**：Sentence-BERT

### 2. 相似度度量

| 度量方式   | 公式    | 适用场景        |
| ---------- | ------- | --------------- |
| 余弦相似度 | cos(θ)  | 方向相似        |
| 欧氏距离   | √(a-b)² | 距离相近        |
| 点积       | a·b     | 方向+ magnitude |

### 3. 索引方法

#### 精确索引

- **暴力搜索**：O(n) 全量比较
- **KD-Tree**：低维 O(log n)

#### 近似索引（ANN）

- **HNSW**：图基，最常用
- **PQ**：量化，压缩率高
- **LSH**：哈希，容错好

## 搜索流程

```
1. 文档 → 向量化模型 → 向量存储
2. 用户查询 → 向量化 → 向量检索 → Top-K 结果
3. 结果返回 → 可选重排序 → 输出
```

## 进阶技术

### 1. 混合检索

结合向量搜索和关键词搜索：

```typescript
async function hybridSearch(query: string) {
  const vectorResults = await vectorStore.search(query, (k = 10));
  const keywordResults = await keywordSearch(query, (k = 10));
  return rerank([...vectorResults, ...keywordResults]);
}
```

### 2. 分层检索

- **粗排**：HNSW 快速召回 Top-100
- **精排**：BERT 重排 Top-10

### 3. 稀疏/稠密混合

- **BM25**：稀疏检索，关键词精准
- **向量**：稠密检索，语义理解
- **ColBERT**：晚交互，兼顾两者

## 使用边界

**何时不用：**

- 数据量小：暴力搜索即可
- 精确匹配：数据库索引足够
- 低维数据：KD-Tree 比 HNSW 更简单

**注意事项：**

- 向量维度：维度越高，维度灾难越严重
- 索引选择：HNSW 均衡，PQ 压缩，LSH 容错
- 相似度选择：余弦 vs 欧氏 vs 点积各有场景
- 过滤条件：向量检索与过滤条件的结合

**面试常考点：**

- 向量检索算法对比（HNSW vs PQ vs KD-Tree）
- 近似最近邻（ANN）vs 精确最近邻（KNN）
- 维度灾难问题
- RAG 中的向量检索优化
- 混合检索（向量+关键词）
