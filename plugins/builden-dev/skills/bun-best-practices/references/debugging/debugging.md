# 调试规范

本规范定义调试手段的优先级和最佳实践。

## 调试手段优先级

当遇到问题时，按以下优先级排查：

| 优先级 | 手段                | 适用场景                           |
| ------ | ------------------- | ---------------------------------- |
| 1      | **日志**            | 了解发生了什么，快速定位阶段和位置 |
| 2      | **测试**            | 能否复现，精准定位到具体行         |
| 3      | **阅读源码**        | 理解逻辑                           |
| 4      | **chrome-devtools** | 前端问题、DOM 检查、网络分析       |

### "不打断流程"的调试方式

- **测试即调试**：把问题转化为测试用例，既能定位又能防止回归
- **日志即断点**：结构化日志替代临时 console.log
- **错误即触发**：统一错误处理，自动记录上下文

## 调试工具

| 工具            | 用途                         |
| --------------- | ---------------------------- |
| chrome-devtools | 前端问题、DOM 检查、网络分析 |
| Playwright MCP  | AI 驱动的浏览器自动化测试    |
| Sentry          | 错误追踪、性能监控           |

## 调试技巧

### 测试辅助

- 把问题转化为测试用例
- 先写失败测试，再写实现
- 使用单元测试快速定位问题

### 源码分析

- 使用 LSP 进行代码跳转
- 追踪数据流，找到值从哪里来
- 查找类似工作代码进行对比

### 浏览器工具

- 使用 chrome-devtools 检查 DOM、网络、Console
- 使用 Performance 面板分析性能
- 使用 Network 面板检查请求
