# Drawer - 抽屉

从页面边缘滑入的浮层面板，适合承载大量内容或复杂交互。

---

## 定义

Drawer（抽屉）是一个从页面边缘滑入的浮层面板，不阻断主内容区的操作，适合承载大量内容或需要并排操作的场景。

---

## 典型场景

- 详情查看（与列表页并排查看）
- 表单填写（不离开当前页面）
- 设置面板
- 侧边导航（移动端）
- 购物车

---

## 核心设计考量

### 1. 不阻断主流程

- 用户可以同时操作主内容和 Drawer
- 适合需要参考其他内容的场景

### 2. 位置选择

| 位置       | 场景               |
| ---------- | ------------------ |
| 右（默认） | 表单、详情、设置   |
| 左         | 导航、菜单         |
| 上/下      | 简短信息、底部操作 |

### 3. 层级管理

- 遮罩覆盖主内容
- Drawer 层级高于遮罩
- 避免多层级 Drawer

---

## 状态定义

| 状态    | 描述       |
| ------- | ---------- |
| closed  | 关闭       |
| opening | 打开动画中 |
| open    | 完全打开   |
| closing | 关闭动画中 |

---

## 交互行为

### 打开

1. 点击触发
2. 遮罩渐入
3. Drawer 从边缘滑入

### 关闭

1. 点击关闭按钮 / 遮罩 / ESC / 点击返回
2. Drawer 滑出
3. 遮罩渐出

---

## 动效规范

### 滑入/滑出

- 持续时间：250-300ms
- 曲线：ease-out（打开）/ ease-in（关闭）
- 距离：100%（完全移出视口）

### 遮罩

- 持续时间：200ms
- 背景：rgba(0, 0, 0, 0.45)

---

## 尺寸规范

| 位置  | 宽度/高度                      |
| ----- | ------------------------------ |
| 左/右 | 380px（移动端）/ 480px（桌面） |
| 上/下 | 30vh / 45vh                    |

---

## 可访问性

- `role="dialog"` 或 `role="drawer"`
- 焦点管理（打开聚焦，关闭还原）
- 键盘支持（ESC 关闭）

---

## Modal vs Drawer

| 特性     | Modal    | Drawer        |
| -------- | -------- | ------------- |
| 阻断操作 | 是       | 否            |
| 适合内容 | 简短确认 | 复杂表单/详情 |
| 适合场景 | 需要专注 | 需要并排操作  |
| 动画     | 缩放     | 滑动          |

---

## 禁忌

- ❌ 重要确认操作（用户可能忽略）
- ❌ 移动端过度使用
- ❌ 内容过于简单（浪费空间）
