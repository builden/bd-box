# Modal - 对话框

用于承载较重的交互内容，需要用户明确确认或操作的组件。

---

## 定义

Modal（对话框）是一个覆盖在主内容之上的浮层，阻断用户与页面其他内容的交互，用于承载重要的操作或信息。

---

## 典型场景

- 表单填写（复杂表单）
- 确认操作（删除、提交）
- 展示详情（需要用户专注的内容）
- 登录/注册

---

## 核心设计考量

### 1. 内容适度

- Modal 适合承载较复杂的内容
- 如果内容简单，考虑使用 Drawer 或 Tooltip
- 避免在 Modal 中嵌套 Modal

### 2. 明确的关闭方式

- 右上角关闭按钮
- 点击遮罩关闭（可配置）
- ESC 键关闭
- 提供取消/确认按钮

### 3. 聚焦管理

- 打开时自动聚焦到第一个可交互元素
- Tab 键只在 Modal 内部循环
- 关闭时焦点回到触发元素

---

## 状态定义

| 状态     | 描述         |
| -------- | ------------ |
| 默认     | 正常显示     |
| loading  | 操作处理中   |
| disabled | 确认按钮禁用 |

---

## 交互行为

### 打开

1. 点击触发按钮
2. 遮罩渐入
3. Modal 内容从上方滑入或缩放出现
4. 聚焦移到 Modal 内

### 关闭

1. 点击关闭按钮 / 遮罩 / ESC
2. 遮罩渐出
3. Modal 内容滑出或缩小消失
4. 焦点回到触发元素

---

## 动效规范

### 进入动画

- 持续时间：200-250ms
- 曲线：ease-out
- 效果：scale(0.95) → scale(1) + opacity(0 → 1)

### 退出动画

- 持续时间：150-200ms
- 曲线：ease-in
- 效果：scale(1) → scale(0.95) + opacity(1 → 0)

### 遮罩

- 持续时间：200ms
- 背景：rgba(0, 0, 0, 0.45)

---

## 可访问性

- `role="dialog"` 角色
- `aria-modal="true"` 属性
- 键盘导航支持（Tab / Shift+Tab / ESC）
- 屏幕阅读器正确读出标题

---

## 尺寸规范

| 场景 | 宽度        |
| ---- | ----------- |
| 小   | 400px       |
| 默认 | 520px       |
| 大   | 720px       |
| 全屏 | 90vw × 90vh |

---

## 禁忌

- ❌ 过多的 Modal 会打断用户流程
- ❌ 过于简单的确认使用 Modal
- ❌ 移动端大量使用 Modal
