# 中文写作强制规范

## 核心原则

| 原则       | 说明                       |
| ---------- | -------------------------- |
| 一句话说清 | 绝不车轱辘话               |
| 拒绝模糊   | 用"点击按钮"而非"进行操作" |
| 能简则简   | 每个词都要有意义           |

## 写作规则

### 1. 删除冗余词句

```diff
- 这个功能主要是用于帮助用户能够更加方便地管理他们的任务
+ 这个功能用于管理任务
```

### 2. 使用肯定句式

```diff
- 不要使用 npm install
- 不能跳过测试
+ 必须使用 bun install
+ 必须运行测试
```

### 3. 使用具体的语言

| 避免     | 使用             |
| -------- | ---------------- |
| 进行操作 | 点击、保存、删除 |
| 若干个   | 3个、5个         |
| 相关     | 具体说明什么相关 |

### 4. 段落结构

- 一段一主题
- 段落首句应是主题句
- 避免长段落（超过 10 行考虑拆分）

### 5. 句式变化

- 并列概念用并列形式：安装 → 运行 → 测试
- 相关词放一起：修饰语靠近被修饰词

## 风格红线

| 禁止               | 原因             |
| ------------------ | ---------------- |
| "非常""极其""相当" | 程度副词稀释表达 |
| "车轱辘话"         | 重复表达无意义   |
| 被动语态           | 中文被动不自然   |
| 浮夸词汇           | 显得不够专业     |

## 示例

### 好的例子

```
安装依赖：
bun install

运行测试：
bun test
```

### 差的例子

```
首先，我们需要使用 bun install 来安装所有的依赖项，这是非常重要的一步。
然后，在安装完成之后，我们需要运行测试来确保代码的正确性。
```
