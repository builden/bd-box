# 评估与迭代

本文档提供构建有效评估和迭代 Skills 的策略。

## 评估驱动开发

在编写大量文档之前先创建评估。这确保你的 Skill 解决的是真实问题而非想象的问题。

### 流程

1. **识别差距**：在没有 Skill 的情况下运行 Agent，记录具体失败或缺失的上下文
2. **创建评估**：构建测试这些差距的三个场景
3. **建立基准**：测量没有 Skill 时 Agent 的表现
4. **编写最小指令**：创建足以解决这些差距并通过评估的内容
5. **迭代**：执行评估，与基准比较并改进

## 评估结构

```json
{
  "skills": ["pdf-processing"],
  "query": "从 PDF 文件提取所有文本并保存到 output.txt",
  "files": ["test-files/document.pdf"],
  "expected_behavior": [
    "成功使用适当的 PDF 处理库或命令行工具读取 PDF 文件",
    "提取文档所有页面的文本内容，不遗漏任何页面",
    "以清晰易读的格式将提取的文本保存到 output.txt 文件"
  ]
}
```

## 迭代开发 Skills

### 创建新 Skill

1. **不使用 Skill 完成一项任务**：使用普通提示词与 Agent 一起完成任务。注意你反复提供的上下文。

2. **识别可复用模式**：完成任务后，识别你提供的对类似未来任务有用的上下文。

3. **让 Agent 创建 Skill**："创建一个 Skill 来捕捉我们刚刚使用的 BigQuery 分析模式。"

4. **审查简洁性**：检查 Agent 是否添加了不必要的解释。

5. **改进信息架构**：让 Agent 更有效地组织内容。

6. **在类似任务上测试**：在新实例中使用 Skill 处理相关用例。

7. **根据观察迭代**：如果 Agent 遇到困难，返回并提供具体信息："当 Agent 使用此 Skill 时，它忘记了按日期过滤..."

### 迭代现有 Skills

1. **在真实工作流中使用 Skill**：给 Agent 实际任务，而非测试场景

2. **观察 Agent 的行为**：注意它在哪里遇到困难、成功或做出意外选择

3. **返回让 Agent 改进**：分享当前的 SKILL.md 并描述你的观察结果。

4. **审查建议**：考虑重新组织以使规则更突出。

5. **应用并测试更改**：更新 Skill，然后再次测试。

6. **根据使用情况重复**：继续这个观察-改进-测试循环。

## 使用不同模型测试

- **快速、经济型模型**：Skill 是否提供足够的指导？
- **平衡型模型**：Skill 是否清晰高效？
- **强推理型模型**：Skill 是否避免了过度解释？

在强大模型上完美的内容可能在快速模型上需要更多细节。如果跨多个模型使用，力求在所有模型上都能良好工作。

## 观察 Agent 如何导航 Skills

注意：

- **意外的探索路径**：Agent 是否以你没想到的顺序读取文件？
- **错过的连接**：Agent 是否未能跟随到重要文件的引用？
- **过度依赖某些部分**：如果 Agent 反复读取同一文件，考虑是否该内容应该放在主 SKILL.md 中
- **被忽略的内容**：如果 Agent 从不访问某个捆绑文件，它可能是不必要的或在主指令中信号不明显

## 收集团队反馈

1. 与团队成员分享 Skills 并观察他们的使用情况
2. 询问：Skill 是否在预期时激活？指令是否清晰？缺少什么？
3. 纳入反馈以解决你自己使用模式中的盲点

## 纪律性 Skills 的专门测试

对于强制执行 discipline 的 Skills（如 TDD、代码审查要求），通用评估方法可能不足。这些 Skills 需要抵抗 Agent 的理性化倾向。

详见 [disciplined-testing skill](../disciplined-testing/SKILL.md)。
