# Design Patterns Skill

设计模式与原则指南，帮助开发者选择合适的模式。

## 使用场景

- "用什么模式好"
- "这段代码用什么设计模式"
- "代码太耦合了"
- "需要扩展"
- 提及具体模式名（策略、观察者、单例等）

## 模式分类

### 经典 23 种

- 创建型：单例、工厂、抽象工厂、建造者、原型
- 结构型：适配器、桥接、组合、装饰器、外观、享元、代理
- 行为型：策略、观察者、命令、迭代器、状态、职责链、模板方法、备忘录、中介者、访问者、解释器

### Modern Patterns (patterns.dev)

- Vanilla JS：模块、原型、Mixin、中介者、观察者、Provider、代理
- React：Hooks、Container/Presentational、Render Props、HOC、Compound Components
- Vue：Components、Composables、Provide/Inject、Dynamic Components
- 渲染模式：SSR、SSG、Streaming SSR、Hydration、RSC
