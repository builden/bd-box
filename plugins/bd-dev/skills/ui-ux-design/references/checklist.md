# 设计检查清单

快速验证 UI 设计是否符合规范的参考清单。

---

## 颜色检查

- [ ] 主色使用品牌色，不超过 3 个
- [ ] 文本色有层级区分（主要/次要/禁用）
- [ ] 功能色符合语义（成功/警告/错误/提示）
- [ ] 暗色模式有正确的颜色映射
- [ ] 对比度满足 WCAG 标准（4.5:1）

---

## 间距检查

- [ ] 使用 8px 基础网格系统
- [ ] 同类元素间距一致
- [ ] 组件内 padding 合理（不过大/过小）
- [ ] 多层次内容有足够的视觉区分

---

## 层级检查

- [ ] 标题/正文/次要文字有层级区分
- [ ] 阴影系统清晰（sm/md/lg/xl）
- [ ] z-index 层级合理（不冲突）
- [ ] 弹层（Modal/Drawer）有足够高 z-index

---

## 状态检查

- [ ] 加载状态有骨架屏或 Spin
- [ ] 空状态有友好提示和操作引导
- [ ] 错误状态有清晰的错误信息和重试操作
- [ ] 所有可交互元素有 hover/focus/active 状态

---

## 组件检查

- [ ] 按钮有明确的主/次/虚线/文字类型
- [ ] 输入框有正确的标签和占位符
- [ ] 表单验证错误有明确的错误提示
- [ ] 弹层有 ESC 关闭和点击遮罩关闭
- [ ] 确认操作有二次确认（Modal）

---

## 动效检查

- [ ] 交互反馈有适当的过渡动画
- [ ] 动画时长合理（150-300ms）
- [ ] 使用 ease-out/ease-in-out 等自然曲线
- [ ] 避免过于花哨的动效
- [ ] 考虑 `prefers-reduced-motion` 用户

---

## 可访问性检查

- [ ] 所有图片有 alt 文本
- [ ] 表单标签正确关联（label for）
- [ ] 焦点状态可见
- [ ] 键盘可以完成所有操作
- [ ] 颜色不是唯一的信息载体

---

## 响应式检查

- [ ] 移动端布局合理（不横向滚动）
- [ ] 触摸目标足够大（≥44px）
- [ ] 文字在移动端可读
- [ ] 断点设置合理

---

## 代码检查

- [ ] 使用 CSS 变量管理主题
- [ ] 避免内联样式
- [ ] 类名命名清晰
- [ ] 重复代码提取为组件
