# 设计价值观

设计价值观是评估设计质量的内在标准，为具体的设计决策提供指导方向。

---

## Ant Design

### 确定性（Certain）

接口是用户与系统交互的媒介，是手段而非目的。在追求"自然"交互的基础上，产品界面应该具有高确定性。

**用户确定性**：

- 单一产品内的一致性
- 跨产品协作的确定性
- 降低用户认知负荷

**设计确定性**：

- 组件 API 的可预测性
- 交互行为的确定性
- 状态管理的明确性

### 一致性（Consistent）

统一的视觉和交互语言，降低用户学习成本。

- **视觉一致性**：色彩、字体、间距、图标风格统一
- **交互一致性**：相似的操作有相似的反馈
- **跨平台一致性**：不同平台保持核心体验一致

### 亲密性（Proximity）

相关内容在空间上靠近，根据格式塔心理学原理组织界面元素。

- 相关的操作按钮放在一起
- 表单标签与输入框靠近
- 标题与内容形成视觉单元

### 反馈（Feedback）

每个操作都有即时响应，让用户时刻了解系统状态。

- **过程反馈**：加载状态、进度指示
- **结果反馈**：操作成功/失败提示
- **输入反馈**：实时校验、错误提示

### 可控（Controllable）

用户始终掌握主导权，可以自由导航和操作。

- 支持撤销/重做
- 提供取消操作
- 允许用户自定义部分体验

## Apple HIG

### 清晰（Clarity）

界面必须易于阅读和导航，内容和功能一目了然。

- 文本在任何尺寸下都清晰可读
- 图标精确、明了
- 装饰最小化，突出功能
- 使用清晰的文案

### 尊重（Deference）

界面永远不应该与内容竞争，应该让内容成为焦点。

- 界面元素服务于内容
- 减少不必要的装饰
- 使用半透明效果处理背景
- 让用户的注意力集中在内容上

### 深度（Depth）

通过视觉层次和动效传达层级关系，帮助用户理解元素之间的关系。

- 使用层级（layers）组织内容
- 利用阴影表达空间关系
- 使用平滑的过渡动画
- 3D Touch / 按压触感增加交互深度

## 价值观对比

| 维度         | Ant Design                    | Apple HIG            |
| ------------ | ----------------------------- | -------------------- |
| **核心目标** | 企业级产品的高效构建          | 极致的用户体验       |
| **确定性**   | 强调组件 API 和交互的可预测性 | 强调界面的可理解性   |
| **一致性**   | 强调跨产品协作的一致          | 强调平台内的体验一致 |
| **反馈**     | 强调操作的即时响应            | 强调状态的可感知性   |
| **可控**     | 强调用户的主动权              | 强调对内容的主导     |

## 应用原则

### 优先级

1. **可用性优先**：确保功能可用的前提下追求美观
2. **一致性其次**：统一的体验降低认知负荷
3. **美观是加分**：在核心体验稳固后追求视觉品质

### 决策框架

当遇到设计决策时，问自己：

- 这个设计是否降低了用户的认知负荷？
- 这个设计是否提供了清晰的反馈？
- 这个设计是否让用户感到可控？
- 这个设计是否与整体风格一致？
