# Toast - 轻提示

非阻断式的全局通知，短暂显示后自动消失。

---

## 定义

Toast（轻提示）是一个非阻断式的通知组件，从页面顶部或中部滑入，显示短暂时间后自动消失，不会打断用户当前操作。

---

## 典型场景

- 操作成功/失败反馈
- 保存成功提示
- 网络状态变化通知
- 系统消息推送

---

## 核心设计考量

### 1. 非阻断

- 用户可以继续当前操作
- 不需要用户交互（点击确认）
- 自动消失

### 2. 简洁明确

- 一句话说明状态
- 避免复杂内容
- 不包含操作按钮

### 3. 显示位置

| 位置         | 场景               |
| ------------ | ------------------ |
| 顶部（默认） | 全局通知、重要消息 |
| 顶部居中     | 重要但不紧急       |
| 底部         | 移动端操作反馈     |

---

## 状态定义

| 状态    | 描述     |
| ------- | -------- |
| info    | 信息提示 |
| success | 成功反馈 |
| warning | 警告提示 |
| error   | 错误反馈 |
| loading | 加载中   |

---

## 交互行为

### 显示

1. 触发显示
2. Toast 滑入
3. 停留指定时间（默认 3s）
4. 自动滑出

### 手动关闭

- 提供关闭按钮（可选）
- 点击关闭立即消失

---

## 动效规范

### 滑入

- 持续时间：200ms
- 曲线：ease-out
- 效果：从上方/下方滑入

### 滑出

- 持续时间：150ms
- 曲线：ease-in
- 效果：向原方向滑出

### 停留

- 默认：3 秒
- 可配置：短（2s）/ 长（5s）

- 最大宽度：400px

---

## 可访问性

- 使用 `role="alert"` 或 `role="status"`
- 屏幕阅读器正确读出
- 支持键盘关闭（可选）

---

## 禁忌

- ❌ 重要操作不要只依赖 Toast
- ❌ 避免频繁显示多条 Toast
- ❌ 避免在 Toast 中放置交互元素
- ❌ 错误信息避免只使用 Toast（重要错误用 Modal）
