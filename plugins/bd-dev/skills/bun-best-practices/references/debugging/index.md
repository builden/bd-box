## 调试原则

1. **不打断流程**：测试即调试、日志即断点、错误即触发
2. **最小复现**：用最少的代码复现问题
3. **一次只改一个变量**：定位问题后逐一排查

---

## 日志核心

- 工具：pino + pino-pretty
- 级别：debug → info → warn → error
- 关键节点：请求入口 / 业务操作 / 外部调用 / 异常捕获 / 请求出口

详见 [logging.md](./logging.md)
